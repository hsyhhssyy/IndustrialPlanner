# S6-RQ-001 Playwright CLI 自动化测试流程

## 文档内容范围、结构与格式约定（必读）

### 1. 本文档负责什么

本文档用于沉淀 S6-RQ-001 上的全部结构化记忆，并采用“状态视图 + 时间线附录”的混合写法，只负责本需求相关的：

1. Playwright CLI 自动化测试流程建设。
2. 自动化范围、工件沉淀、执行口径与回归入口。
3. 测试脚本演进、失败定位与修复记录。
4. 作为 Stage5 顺延项迁入 Stage6 后的新阶段执行口径。

本文档不负责：

1. 代替其他需求保存其全部测试结论。
2. 记录一次性手工测试的所有细节。
3. 记录逐字聊天内容。

### 2. 本文档固定结构

1. 文档内容范围、结构与格式约定
2. 需求卡片
3. 背景与目标
4. 当前口径
5. 当前状态视图
6. 自动化范围
7. 验收标准
8. 测试记录
9. bug / 修复记录
10. 中断与恢复记录
11. 需求拉扯与决策时间线
12. 待确认问题
13. 状态更新

### 3. 本文档记录格式

1. 文档前半部分优先说明当前自动化覆盖状态、当前阻塞与下一步。
2. 自动化范围至少标明“已覆盖 / 未覆盖 / 暂缓”。
3. 每次自动化执行记录至少写明：时间、入口、结果、失败点、工件位置。
4. 测试工件目录变化时，必须同步更新本文档。
5. 自动化执行过程中发现的跨需求 bug，同步登记到 [../bugs.md](../bugs.md)。

## 背景与目标

Stage5 已明确 Playwright CLI 自动化测试流程不纳入其发版闭环，因此 Stage6 接手该需求，目标是在稳定阶段内建立真正可执行、可回归、可沉淀的自动化测试流程，使 AI 可以稳定重复执行启动冒烟、核心主流程、关键新功能与已修复缺陷回归。

## 当前口径

1. 自动化测试基于 Playwright CLI。
2. 测试工件、截图与脚本统一沉淀到 `.temp/playwright-cli-test/` 规范目录。
3. 自动化范围至少覆盖：
   - 启动冒烟；
   - 核心仿真链路；
   - 1 个已发布高价值功能回归；
   - 1 个已修复缺陷回归。
4. 测试失败时必须能定位到具体步骤或截图。
5. Stage6 中该需求优先服务稳定性回归，而不是为覆盖率单独膨胀范围。
