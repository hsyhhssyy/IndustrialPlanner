## 背景与目标

当前用户已明确要求将“移动端兼容”拆分为“竖屏显示”和“触控支持”两条需求。`S6-RQ-002` 只处理竖屏场景下的界面逻辑调整，不以设备类型区分，而以视口比例判断是否进入竖屏口径。该需求的第一步基线包括：恢复竖屏检测能力，并在底部状态栏显示“移动端”，用于确认当前已进入竖屏适配逻辑。

## 当前口径

1. “竖屏”定义为：`视口宽度 < 视口高度 * 3/4`。
2. 本需求只处理 UI 逻辑与布局表现，不负责触控操作差异。
3. 竖屏下允许组件重新排布，例如：竖排按钮改为横排、栏位折叠策略变化、信息密度调整。
4. 底部状态栏应提供竖屏监测提示，当前用户期望的提示文案为“移动端”。
5. pad 与折叠屏手机只要满足竖屏比例定义，也属于本需求覆盖范围。
