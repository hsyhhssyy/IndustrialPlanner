# Stage2 开发文档（冻结版）

本文档集已按当前代码实现状态冻结，作为 Stage2 交付基线。

## 冻结时间

- 冻结日期：2026-02-26
- 冻结依据：当前 `main` 分支代码行为

## Stage2 冻结范围（已实现）

- 产线规划器入口、浮窗、目标输入、多目标聚合、配方切换重算。
- 地区切换（四号谷地/武陵）与武陵内容过滤。
- 条件配方启用（蓝铁块/蓝铁粉末互转）与农业闭环平衡修正。
- 武陵流体基础：管道、管道分流/汇流/桥接、流体端口与流体配方接入。
- 固液介质隔离与端口类型约束（液体仅管道、固体仅传送带）。
- 武陵扩展区域放置规则（抽水泵仅外圈、部分流体设施可外圈放置）。
- 武陵特定建筑数量上限（含天有烘炉上限拦截）。
- 反应池可配置（最多两条配方、输出路由、并行推进）。
- 水培设备独立建模并接入系统。
- Toast 降噪策略已落地（高频状态切换不做强提示）。

## Stage2 冻结口径（与早期草案差异）

- 反应池当前为“5 个共享槽位”实现口径（非“3 固体 + 2 液体独立槽位”）。
- 抽水泵当前支持多液体选择输出（不仅限清水）。
- 电力当前仍为“接网即运行前置”口径，未引入完整功率统计模型。

## 已移交 Stage3 的未实现项

以下需求已从 Stage2 移出，统一纳入 Stage3：

- 全设备 `powerDemand/powerSupply` 建模与展示。
- 电力三窗口统计（即时、10 分钟、1 小时）按标准秒滚动。
- 电力不足不降速不停机（与当前 `NO_POWER` 停机逻辑解耦）。
- 管道“等效缓存池容量 = 2 × 连通长度”模型。
- 管道“单液体锁定/排空后切换”传染规则。
- 抽水泵“输出效率来自领域模型配置”的专门吞吐口径。
- 瓶装液体“隐藏组合项 + 专用选择器 + 单展示多展开”完整方案。
- 反应池“3 固体 + 2 液体独立槽位”模型对齐。

详见 Stage3 文档：`../stage3/README.md`。
