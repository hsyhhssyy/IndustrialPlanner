# Stage2 发布说明

## 1. 版本信息

- 版本号：Stage2-v0（文档初版）
- 发布日期：2026-02-23
- 发布负责人：待填

## 2. 本次发布内容

### 2.1 新增

- 新增“产线规划器”入口与浮窗交互。
- 支持多目标吞吐输入与按物品聚合结果展示。
- 支持多配方手动切换与全局重算。
- 新增地区能力：四号谷地/武陵双地区可用。
- 武陵地区引入流体工业三要素（管道铺设、管道端口设备、流体配方）。
- 管道本体采用有方向输送规则：需按端口方向输送液体。
- 管道分流器/汇流器采用有方向规则：方向不匹配时不运输。
- 连通管道本体按等效管道缓存池（抽象模型，非设备“储液罐”）结算：容量为 `2 × 管道长度`。
- 真实设备“储液罐”保持定向语义：`1` 入口 + `1` 出口。
- 武陵扩展区域允许布置流体设施（抽水泵、管道、储水罐、管路设备）。
- 新增抽水泵：作为清水无限资源入口，可通过管道引入基地。
- 修正物品分类：紫晶矿补充为矿物（固体）分类。
- 新增瓶装液体体系：内部提供 20 个隐藏组合项（4 瓶型 × 5 液体）并通过专用选择器交互。
- 新增武陵特定建筑摆放上限：如天有烘炉最多放置 `N` 个。
- 引入管道传染规则：同一管道系统仅允许一种液体，异液输入在排空前阻塞。
- 启用武陵特殊设备“反应池”：支持多配方并行与产物缓存路由。
- 形态切换类能力采用独立设备实现：新增水培设备（如“种植机（水培）”）。
- 引入全设备电力属性展示（耗电/发电），用于发电调试。
- 新增电力统计三组数据：即时值、10 分钟滚动均值、1 小时滚动均值。

### 2.2 优化

- 结果区采用三列伪表格卡片（物品/配方、需求、设备）。
- 输入吞吐信息下沉到第一列次信息区域，减少视觉重复。
- Toast 提示治理：高频显式操作默认静默，降低噪音。

### 2.3 规则修正与口径明确

- 蓝铁块/蓝铁粉末互转配方改为条件启用（仅双需求可选）。
- 种植机/采种机农业闭环按平衡口径计算，不显示循环异常提示。
- 电力不足不触发设备停机或降速；“必须接入电网”前置约束保持不变。
- 电力统计时间基准为标准秒，不使用现实时间。
- 桥接器按双通道独立处理，传染不跨通道传播。
- 管道系统支持双端提交/抽取：同一连通管道本体可在任意端注入与取出液体。
- 反应池未选配方时不生产，产物由用户指定缓存对应端口送出。
- 水培设备不支持运行时形态切换，需以独立设备放置与运行。
- 抽水泵仅允许放置在扩展区域，基地内放置视为非法。
- 紫晶矿按固体侧运输与端口规则处理，不参与液体链路。
- 瓶型无关配方在 UI 仅显示 1 条，执行层按瓶型展开隐藏配方并按输入瓶型返瓶。
- 武陵特定建筑达到上限后禁止继续放置，删除后恢复可用名额。

## 3. 升级影响

- 对用户影响：中（新增规划流程，但不改变 Stage1 主流程）。
- 对配置影响：有（新增规划器状态与偏好持久化）。
- 对历史数据影响：无（Stage1 核心数据口径保持兼容）。

## 4. 已知限制

- 目前不支持自动布局、自动连线与最优解求解。
- 目前不支持复杂流体网络求解（压力、温度、相变）。
- 规划结果不直接驱动地图自动摆放。
- 当前不实现电力潮流网络求解与按缺电惩罚生产。
- 当前不模拟基地外完整世界行为；扩展区域仅承载抽水入口语义。
