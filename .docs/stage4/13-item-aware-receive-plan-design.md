# Stage4 专项设计：下游接收前置判定 item-aware 化（S4-RQ-003）

## 1. 目标

- 将发送规划阶段的前置门禁判定从“仅车道状态”升级为“车道状态 + 待传输物品ID”联合判定。
- 保证过滤类物流设备可在前置阶段准确报告对当前物品的接收能力。

## 2. 背景问题

- 现状中，`canReceiveOnPortWithPlan` 主要基于 lane 占用与时序返回 `canTry/canAccept`。
- 物品维度通常在后续 `canAcceptIntoLane(..., itemId)` 才参与判断，造成前置判定与最终接收判定口径分离。

## 3. 设计决策

1. 采用方案A：`canReceiveOnPortWithPlan` 增加 `itemId`（及必要上下文）参数。
2. 前置判定阶段即可针对目标设备执行 item-aware 接收能力判断。
3. 本需求独立编号与追踪，不作为物品准入口/管道准入口附属需求。

## 4. 约束

- 不改变既有 0.5 阈值与三态调度骨架（`canTry/canAccept` 机制保留）。
- 不改变无物品特化规则设备的既有行为。
- 对无法在前置阶段完整判定的场景，需保持与后续最终校验一致且可解释。

## 5. 验证点

- 同一 tick 内，前置 `canAccept` 与最终接收校验在关键场景下保持一致。
- 过滤类物流设备对“允许物品 / 非允许物品 / 计数后封堵”返回符合预期。
- 不出现新增死锁、饥饿或异常吞吐抖动。
