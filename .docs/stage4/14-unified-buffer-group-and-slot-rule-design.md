# Stage4 专项设计：统一缓冲组与槽位顺序规则（S4-RQ-004）

## 1. 目标

- 统一多输入/多输出设备的端口选择与缓冲接收/输出机制。
- 将协议储存箱、普通设备、反应池收敛到同一套“缓冲组 + 槽位模式 + 顺序检索”框架。

## 2. 核心模型

### 2.1 缓冲组（Buffer Group）

1. 单设备可包含多组端口-缓冲区映射。
2. 每组独立维护：
   - 输入端口集合
   - 输出端口集合
   - 目标槽位集合
   - 组内 RR 游标（用于 `orderedInLinks`）
   - 组内 RR 游标（用于 `orderedOutLinks`）
3. 各组分开搜索与仲裁，不共享组内游标。

### 2.2 槽位（Slot）

每个槽位包含：

- `slotIndex`：顺序位置
- `mode`：`free | pinned`
- `pinnedItemId`：仅在 `pinned` 模式下有效
- `amount / capacity`
- `currentItemId`（空槽可为空）

## 3. 输入规则（统一）

1. 先在缓冲组内运行 `orderedInLinks`，选择端口尝试顺序（RR）。
2. 对当前来料，在该组槽位内按 `slotIndex` 执行 `sequential` 扫描。
3. 命中首个可接收槽位即放入，不继续扫描。

### 3.1 槽位可接收判定

- `free`：
  - 空槽：可接收符合组筛选的任意物品。
  - 非空：仅接收与 `currentItemId` 相同物品（堆叠到满）。
- `pinned`：
  - 仅当来料等于 `pinnedItemId` 时可接收。
  - 即使空槽，也按“已绑定可堆叠位”处理，不接受非绑定物品。

### 3.2 汇流器输入优先级策略（与 S4-RQ-004 一体）

1. 在汇流器的 `orderedInLinks` 中，分流器来源端口优先级高于传送带来源端口。
2. 若存在至少一个“分流器来源且可输入”的端口，则仅在该优先集合内进行 RR 选择。
3. 仅当分流器来源端口全部不可输入时，才回退到传送带来源端口集合进行 RR 选择。
4. 该优先级策略与 RR 组合关系为：来源类型分层优先 > 层内 RR。

## 4. 输出规则（统一）

1. 在缓冲组内按槽位顺序（`slotIndex`）从前到后选择候选输出槽位。
2. 取到候选槽位物品后，运行该组 `orderedOutLinks` 获取端口尝试顺序（RR）。
3. 按 `orderedOutLinks` 顺序逐端口校验下游接收能力。
4. 若某端口可接收，则从当前候选槽位输出并结束本次输出决策。
5. 若该候选槽位在本轮所有输出端口均不可接收，则跳到下一个槽位继续判断。
6. 若全部槽位均不可输出，则本轮该组不发生输出。

### 4.1 输出决策优先级

- 第一优先级：槽位顺序（前槽位先判定）。
- 第二优先级：端口顺序（`orderedOutLinks` RR）。
- 回退策略：前槽位“全端口不可达”时，允许降级尝试后槽位。

## 5. 设备实例化约定

### 5.1 协议储存箱（关联 S4-RQ-002）

- 使用本机制实现，不再单独定义一套输入判定分支。
- 实例配置：单缓冲组、6 槽位、容量 50、槽位顺序固定、槽位模式均为 `free`。
- S4-RQ-002 的输入/输出顺序规则以本机制为准：
  - 输入：`orderedInLinks` + `sequential`（首个可接收槽位）。
  - 输出：槽位顺序优先 + `orderedOutLinks`，全端口不可出时跳到下一个槽位。

### 5.2 普通设备

- 常见形态为“多输入端口 -> 单缓冲组（少量槽位）”。
- 槽位多采用 `free`，自然形成“空槽接收任意、占槽后同类堆叠”的行为。

### 5.3 反应池

- 可配置为多组映射，且槽位多采用 `pinned`。
- 每个 `pinned` 槽位的绑定物品由用户配置。

## 6. 约束

- 不改变既有 0.5 节奏与三态门禁框架。
- 分组仲裁相互独立，禁止跨组共享游标造成串扰。
- 输入规则冲突时，以“组筛选 -> 槽位模式 -> 顺序优先”执行。
- 输出规则冲突时，以“槽位顺序 -> `orderedOutLinks` -> 槽位跳转”执行。
- 推进守恒约束：只有在“链路被仲裁选中且本轮传输提交成功”时，上游输送进度才可推进；未选中或提交失败必须保持原进度。

## 7. 验证点

1. 多组缓冲区设备在并行输入时，组内 RR 正确推进且互不影响。
2. `free` 与 `pinned` 在空槽/非空槽场景行为正确。
3. `sequential` 命中首个可接收槽位并立即停止。
4. 输出满足“槽位顺序优先 + `orderedOutLinks` + 全端口不可达时槽位跳转”。
5. 协议储存箱行为与 S4-RQ-002 需求完全对齐。
6. 汇流器输入满足“分流器来源优先；分流器可用时不选择传送带来源；分流器不可用时回退传送带来源”。
7. 未被选中/未成功提交的上游链路连续多 tick 不会错误推进至终点。
