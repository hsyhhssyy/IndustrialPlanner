# Stage4 专项设计：协议储存箱六格库存化

## 1. 目标

- 将协议储存箱从“仅仿真时提交物品”升级为“具备真实库存与输入输出规则”的设备行为。
- 具体实现机制复用 S4-RQ-004（统一缓冲组与槽位顺序规则）。

## 2. 库存模型

1. 协议储存箱包含固定 6 个格子。
2. 每个格子容量上限 50。
3. 每个格子仅持有一种物品类型（空格视为无物品）。
4. 槽位模式均为 `free`（无 `pinned` 槽位）。

## 3. 初始化规则

- 仿真开始时，支持向协议储存箱置入初始物品。
- 初始置入同样遵循本文件定义的输入规则与容量约束。

## 4. 输出规则

1. 协议储存箱每次输出 1 个物品。
2. 输出先按槽位顺序固定检索（第 1 格到第 6 格）选择候选槽位。
3. 候选槽位确定后，按组内 `orderedOutLinks` 端口顺序逐个尝试下游接收。
4. 若候选槽位在全部输出端口均不可达，则跳到下一槽位继续判定。
5. 命中首个“槽位可出 + 存在可达输出端口”的组合后，从该格扣减 1 并输出。

## 5. 输入规则

1. 输入检索顺序固定为第 1 格到第 6 格。
2. 输入采用 `sequential`：按顺序命中首个可接收槽位即放入。
3. `free` 槽位在空位可接收，非空仅允许同类继续堆叠。
4. 因按顺序选择首个可接收槽位，空位与同类不做额外优先级加权。

## 6. 验证点

- 开局置入后库存状态与配置一致。
- 输出顺序满足“首个非空格优先”。
- 输入顺序满足“首个可放置格优先（空位/同类同权）”。
- 容量边界满足单格上限 50，不出现溢出。
