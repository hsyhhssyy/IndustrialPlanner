# Stage4 发布说明（草案）

## 版本信息

- 阶段：Stage4
- 状态：代码实现完成，回归收尾中
- 启动日期：2026-03-05

## 面向用户的变化（当前）

1. 设备详情新增功率字段展示（`powerDemand`，单位 `kW`）。
2. 发电设备在详情中显示 `powerDemand=0`。
3. 协议储存箱支持仿真开始时置入初始物品。
4. 协议储存箱采用 6 格库存模型（每格容量 50），并按第 1 格到第 6 格顺序处理输入与输出。
5. 协议储存箱槽位配置对话框已重构为图形化六格编辑器：左侧 2×3 槽位总览，右侧当前槽位编辑，并统一复用通用物品选择器。
6. 下游接收前置判定支持 item-aware（`canReceiveOnPortWithPlan` 可透传并判定当前物品）。
7. 物流仿真已统一到缓冲组/RR/槽位顺序机制，协议储存箱与反应池复用同一组规则。
8. 传送带改为 0→1 缓冲运输语义：上游/下游判定与处理器式输入输出缓冲对齐。
9. world 渲染已升级为 underlay / transit / overlay / adornment 四层，支持 belt 小盒子、pipe 流体、splitter/converger 内部运输动画的全局分层。
10. splitter / converger 增加内部运输动画，并在阻塞时提供卡住态视觉反馈。
11. 产线规划器新增“基础供给优先级”规则：被标记为基础供给的资源不再默认反推到副产物来源，副产物流入仅用于抵扣实际缺口。
12. 产线规划器新增“废弃物流强制处置”规则：指定废弃物在存在产出时必须追加处置节点，并在流程图与机器汇总中展示对应设备数量。

## 升级注意事项

- Stage4 当前包含展示层增强、统一物流缓冲仲裁增强、协议储存箱/反应池规则统一化与运输渲染分层增强，不涉及存档迁移。
- 若后续继续扩充产线规划器的特殊物料规则，应优先采用“基础供给 / 可循环中间产物 / 强制处置废弃物”三类口径归档，避免将副产物流错误视为默认供给来源。

## 已知限制

- 自动化测试与长期回归基线尚未补齐，当前仍以构建通过与手工场景回归为主。
- splitter / converger 的内部运输动画当前仍未直接消费“本 tick 真实计划目的端口”，后续若调整 RR 语义，建议同步补上真实目标端口观测数据。
