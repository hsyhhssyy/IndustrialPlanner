# 操作三：蓝图放置（Blueprint Placement）

## 目的
验证“重置环境 -> 工作区域探测 -> 蓝图放置”的完整链路，并将“精选荞愈胶囊”蓝图成功放置到基地左上角，且贴着上方总线。

## 前置要求
- 本操作前必须先完成：
  1. 重置环境（Environment Reset）
  2. 工作区域探测（Work Area Probing）
- 必须读取“工作区域探测”直接打印并记录的结果：`finalX`、`finalY`、`stepPx`、`rightLegalSteps`、`downLegalSteps`。
- 进入本操作前必须通过 `12px` 准入检查；若当前缩放值不是 `12px`，必须先执行“重置环境 + 工作区域探测”后再进入本操作。
- 进入本操作前必须检查协议核心位置；若协议核心仍在建造区域左上角（处于左上区域即可，不要求精确到单格），必须先执行“重置环境 + 工作区域探测”后再进入本操作。
- 全流程必须在 `1920x1080` 视口下执行。
- 禁止将流程打包为 `.sh` 一次性执行，必须逐步执行并逐步验证。

## 执行纪律（强制）
1. 每一步执行后都必须立即截图。
2. 蓝图放置时必须点击“放置蓝图”按钮（不是“放置模式”按钮）。
3. 蓝图若无法摆放会出现红色边框；需按 1 格粒度移动，直到可放置。
4. 目标落点要求：基地左上角区域，且贴着上方总线。
5. 所有截图与网页工件统一放在：
   - `.temp/playwright-cli-test/[时间戳子目录]/03-blueprint-placement/`
6. 本操作开始前必须完成并留证 `12px` 准入检查。
7. 实际执行 `playwright-cli` 命令时，禁止在命令行使用 Linux 环境变量，命令参数需使用完整绝对路径。
8. `run-code` 使用 `$(cat /完整绝对路径/脚本文件)` 的命令替换是允许的；要求 `cat` 的文件名必须是完整绝对路径。
9. 执行 `run-code` 时如需脚本文件，必须先将脚本文件直接创建到 `.temp/playwright-cli-test/[子文件夹]/[对应任务目录]/`；禁止使用管道命令创建脚本文件。
10. `playwright-cli run-code` 的页面脚本中禁止使用 `process.env`；脚本内如需读写路径，必须使用完整绝对路径。
11. 执行 `run-code` 必须使用脚本文件形式（如 `run-code "$(cat /完整绝对路径/xxx.js)"`）；禁止在命令后直接写内联脚本。
12. toast 监测采用“点击 toast 本体关闭”的方式；**禁止使用 `Esc` 关闭**（会导致退出蓝图模式）。
13. 每次 toast 判定都必须执行固定流程：记录当前鼠标位置 -> 读取 toast 文案 -> 点击 toast 本体关闭 -> 截图 -> 鼠标移回原位 -> 再截图后继续下一步。

## 异常回退策略（强制）
若出现以下任一情况，必须中断当前步骤并回退到链路起点：
- 页面变为 `about:blank`
- 任何无法继续执行的状态（元素不可交互、流程中断且无法就地恢复）
- 进入检查发现当前缩放值不是 `12px`
- 进入检查发现协议核心仍在建造区域左上角

回退动作：
1. 重新执行“重置环境”。
2. 重新执行“工作区域探测”。
3. 再次执行本蓝图放置流程。

## 标准步骤（每一步必须截图）
1. 校验页面可继续（URL 非 `about:blank`）。
2. 执行 `12px` 准入检查并截图。
3. 若当前缩放值不是 `12px`：中断本流程，先执行“重置环境 + 工作区域探测”，然后从本步骤 1 重新开始。
4. 检查协议核心是否仍在建造区域左上角（左上区域判定即可）并截图。
5. 若协议核心仍在建造区域左上角：中断本流程，先执行“重置环境 + 工作区域探测”，然后从本步骤 1 重新开始。
6. 点击“蓝图模式”切换到蓝图模式。
7. 选中“精选荞愈胶囊”蓝图。
8. 点击“放置蓝图”按钮进入放置预览态。
9. 基于“工作区域探测”结果计算工作区中点（由探测区域宽高与基准点换算）作为第一次尝试点，并将蓝图移动到该中点。
10. **第一次尝试在左键触发一次 toast 判定后必须立即截图**，并按“toast 固定流程”执行关闭与回位。
11. 若 toast 为“位置不合法，超出可建造区域”：
  - 先执行“向左上小步进搜索”，最多 `50` 格（每步建议 `1格=stepPx`，必要时可减小步进），每步都要触发 toast 判定。
  - 若 `50` 格后仍为“超出可建造区域”，回到中点，再执行“向右下小步进搜索”最多 `50` 格，并持续 toast 判定。
  - 两段搜索的目标都是让 toast 首次变为“取货口/存货口必须紧贴存取线基段放置”（即已进入建造区但未贴顶）。
  - 每一步 toast 判定后都必须执行“点击 toast 关闭 -> 截图 -> 鼠标回位 -> 截图”。
12. 当 toast 进入“取货口/存货口必须紧贴存取线基段放置”后：
  - 向左小增量步进，直到再次出现“位置不合法，超出可建造区域”；
  - 向右回退一步，使 toast 回到“取货口/存货口必须紧贴存取线基段放置”，据此确认左侧贴边基准。
  - 本阶段每次 toast 出现后同样执行“点击 toast 关闭 -> 截图 -> 鼠标回位 -> 截图”。
13. 在左侧贴边基准下持续向上小幅步进，直到预览变白可放置；若途中 toast 又变为“位置不合法，超出可建造区域”，先回退再减小步进幅度继续。
  - 若该阶段出现 toast，仍按固定流程关闭并回位后再继续。
14. 命中可放置状态后立即点击完成放置，并截图。
15. 放置后确认已成功落位（不再红框拒绝态），并截图。

### toast 固定流程（每次判定后必须执行）

1. 记录当前鼠标坐标（记为 `cursorX/cursorY`）。
2. 左键触发 toast 并读取文案用于判定分支。
3. 点击 toast 本体使其立即关闭（不是点击关闭按钮；toast 本体可点击关闭）。
4. 截图留存“toast 已关闭”状态。
5. 鼠标移动回 `cursorX/cursorY`。
6. 再截图留存“鼠标回位后”状态。
7. 禁止使用 `Esc` 关闭 toast，以免取消蓝图模式。

## 验收标准
- 成功进入蓝图模式。
- 成功选中“精选荞愈胶囊”并点击“放置蓝图”。
- 蓝图最终成功放置在基地左上角且贴上方总线。
- 全流程每一步均有截图可追溯。
- 进入本操作时 `12px` 准入检查通过；若未通过，已按规则执行“重置环境 + 工作区域探测”后再进入。
- 进入本操作时协议核心位置检查通过；若核心仍在建造区域左上角，已按规则执行“重置环境 + 工作区域探测”后再进入。
- 已执行“中点首试（强制截图）-> 左上最多50格 -> 回中点右下最多50格”的区域内准入搜索，且 toast 口径符合规则。
- 所有 toast 判定步骤均执行了“点击 toast 关闭 + 鼠标回位”，并在两处状态（关闭后、回位后）完成截图留存。
