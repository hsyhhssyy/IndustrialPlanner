# Playwright-CLI 测试文档索引

## 全局约束

- **在 `playwright-cli` 中，任何操作步骤都必须截图**。
- 涉及 toast 判定时，必须执行固定流程：记录鼠标位置 -> 读取 toast 文案 -> 点击 toast 本体关闭 -> 截图 -> 鼠标回位 -> 再截图。
- toast 关闭方式必须为“点击 toast 本体”；禁止使用 `Esc`（会导致退出当前交互模式，如蓝图模式）。
- 基地切换是用户态操作，多个基地会分别保存位置数据，不可假设每次都在同一初始点。
- **禁止将多步测试打包为 `.sh` 一次性执行**；AI 必须逐步执行，每一步执行后立即验证结果（状态/坐标/缩放/截图）再进入下一步。
- localStorage 清空验证以“`clear()` 后立即为空”为准；刷新可能自动回写默认键，不能以“刷新后仍为空”作为验收标准。
- “重置环境”缩放验收必须检测到 `12px`；未检测到 `12px` 不得判定通过。
- 任一操作在“进入执行前”必须进行 `12px` 准入检查；若当前缩放值不是 `12px`，必须先执行“重置环境 + 工作区域探测”后再进入目标操作。
- 测试用例执行过程中的网页工件文件（如 `snapshot/yaml/json`）和截图图片，必须统一放在 `.temp/playwright-cli-test/[某个子文件夹]/` 下，且**子文件夹名称必须包含时间戳**。
- 执行 `run-code` 时，如需脚本文件，必须先将脚本文件写入 `.temp/playwright-cli-test/[子文件夹]/[对应任务目录]/`；禁止使用管道命令创建脚本文件，应使用编辑技能直接创建文件。
- 执行 `run-code` 必须使用“脚本文件形式”，禁止在命令后直接内联脚本内容。
- 在 `playwright-cli run-code` 的页面脚本中，禁止使用 `process.env`；脚本中涉及路径时必须写完整绝对路径，禁止相对路径与环境变量拼接路径。
- 实际执行任何 `playwright-cli` 命令时，禁止在命令行中使用 Linux 环境变量（如 `CASE_DIR`、`CASE_ROOT`、`$CASE_DIR`）；必须使用完整绝对路径。
- `run-code` 后使用 `$(cat /完整绝对路径/脚本文件)` 这种命令替换是允许的；要求是 `cat` 的文件名必须为完整绝对路径。
- **所有测试必须在 1920x1080 视口下执行**（进入用例后的首批操作必须包含 `playwright-cli resize 1920 1080`，并截图留证）。

## 命名规范

- 推荐时间戳格式：`YYYY-MM-DD_HH-mm-ss`
- 推荐子目录格式：`YYYY-MM-DD_HH-mm-ss_<operation-name>`
- 示例：`2026-03-06_07-35-00_work-area-probing`

## 示例操作目录

- [重置环境](example_operations/01-environment-reset.md)
  - 用于统一测试环境：清空 localStorage、重开页面、缩放到最小。
  - 属于非幂等环境重设操作，应在每条测试链起始执行一次，不应在每个操作开始前重复执行。

- [工作区域探测](example_operations/02-work-area-probing.md)
  - 用于逐格探测可建造区域大小（含 `9x9` 尺寸换算）并与 UI 比对。
  - 包含“先大步接近右下角，再小步四向试探，最后邻域三探测后松手”的完整流程。

- [蓝图放置](example_operations/03-blueprint-placement.md)
  - 用于验证蓝图放置流程（进入放置态、合法落位、非法拒绝）。
  - 作为首个正式测试用例模板，要求先执行“重置环境 + 工作区域探测”再进入本用例。

## 推荐使用方式

1. 先看索引确认要执行的操作。
2. 每条测试链开始时，先执行一次“重置环境”，再完成“工作区域探测”等后续步骤；不要在每个操作开始前重复执行重置。
3. 在进入任一具体操作前执行 `12px` 准入检查；若不满足 `12px`，先执行“重置环境 + 工作区域探测”，再继续目标操作。
4. 进入对应文档按步骤执行并保存截图。
5. 切换基地时优先在基地内归一化；仅在开启新测试链或需要全局清理时执行“重置环境”。
