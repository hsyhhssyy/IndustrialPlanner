# 迁移过程设计指南

> 适用范围：Stage5 及后续所有涉及“旧数据替换 / 正式资源收口 / 历史兼容清理”的需求。

## 1. 文档目标

本文件用于给 AI 和人类提供统一的迁移设计原则，避免在每次替换物品、设备、资源或持久化结构时重新讨论同一套默认规则。

本文件只回答三个问题：

1. 什么时候应该做迁移；
2. 什么时候不应该做迁移；
3. 迁移实现时应遵循什么顺序与边界。

## 2. 迁移设计总原则

1. **正式口径优先**：一旦真实 1.1 数据、正式资源或正式 ID 已确认，应优先收口到正式口径，不保留长期双轨。
2. **单向收敛优先**：优先采用“新口径替换旧口径”的单向流程，而不是引入长期别名层。
3. **未知数据保守处理**：历史数据里读到未知内容时，优先删除或清空，不主动猜测映射关系。
4. **资源类型严格分离**：物品资源、设备图标、设备精灵分开管理，不混写路径。
5. **UI 过滤默认保守**：为降低噪音允许增加辅助开关，但默认值应尽量不改变用户当前认知。
6. **文档回写同步进行**：迁移规则一旦形成，应同步更新需求文档、FAQ、索引与总览入口。

## 3. 推荐迁移流程

### 3.1 物品迁移

适用于“旧物品 ID 被正式物品替换”的场景。

推荐顺序：

1. 新增正式物品定义；
2. 补齐正式物品文案；
3. 补齐正式物品图标；
4. 替换配方、规划器、特殊规则中的旧物品引用；
5. 删除旧物品定义与文案；
6. 将历史持久化里出现的未知物品统一清空为“未配置”；
7. 回归验证关键界面与导入 / 存档链路。

### 3.2 设备迁移

适用于“旧设备逐步退出”或“正式设备资源替换”的场景。

推荐顺序：

1. 确认设备类型 ID 是否仍然有效；
2. 若设备仍有效，只替换资源与引用，不额外制造兼容层；
3. 若旧设备 ID 已不再受支持，则依赖“未知设备删除”规则处理历史数据；
4. 回归验证布局读取、蓝图持久化与导入行为。

### 3.3 UI 过滤迁移

适用于“数据切换期间，物品列表噪音显著上升”的场景。

推荐顺序：

1. 先确认是否属于列表噪音问题，而不是数据缺失问题；
2. 如果需要增加过滤开关，放在用户最接近列表的位置；
3. 默认值保持关闭，避免突然改变既有可见性；
4. 过滤规则只隐藏噪音项，不改变实际数据定义。

## 4. 明确禁止的迁移做法

以下做法默认禁止，除非需求文档明确要求：

1. 为旧 `chrono` 内容长期保留兼容别名；
2. 为未知设备补写“旧 ID -> 新 ID”自动迁移映射；
3. 把设备资源误放到 `itemicon`；
4. 用模糊规则猜测未知物品或未知设备应该映射到什么对象；
5. 为了兼容历史数据而继续保留已经决定废弃的过渡模式。

## 5. 当前 Stage5 已确认规则

### 5.1 未知物品处理

1. 如果持久化配置中出现系统不认识的物品 ID，直接回退为“未配置物品”。
2. 不为旧 `chrono` 物品保留长期兼容映射。
3. 该规则适用于布局配置、规划器配置及其他持久化物品选择状态。

### 5.2 未知设备处理

1. 如果历史布局中出现未知设备，直接忽略并删除。
2. 如果持久化蓝图中出现未知设备，直接忽略并删除。
3. 如果导入蓝图时出现未知设备，直接忽略并删除。
4. 默认不新增旧设备 ID 到新设备 ID 的兼容迁移。

### 5.3 资源路径规则

1. 物品图标：`public/itemicon/<itemId>.png`
2. 设备图标：`public/device-icons/<deviceTypeId>.png`
3. 设备精灵：`public/sprites/<deviceTypeId>.png`
4. 设备不使用 `itemicon`。
5. 如果设备源图四周存在低透明度像素，不能只依赖默认 `trim()`；应按合适的 alpha 阈值裁切后再落盘，避免世界层精灵看起来仍带大面积透明边框。

### 5.4 当前已登记的 UI 辅助过滤

1. 物品选择器“全部物品”分组标题右侧新增“隐藏瓶装液体”开关。
2. 默认值为关闭。
3. 开启后，只在物品列表中隐藏所有瓶装液体条目。
4. 该开关属于列表降噪手段，不改变物品定义与配方数据。
5. 该开关的布局必须独立于通用行内 switch 样式，不能因为继承“整行拉伸”行为而导致控件错位或遮挡。

### 5.5 当前已登记的正式设备收口

1. `item_liquid_cleaner_1` 作为正式设备保留，资源使用独立 `device-icon` 与 `sprite`，并归入“基础生产”。
2. 原“暗管出口”正式 ID 收口为 `item_port_udpipe_unloader_1`。
3. `item_port_udpipe_unloader_1` 的行为口径与抽水泵一致，但允许在建造区内外放置。
4. 这类“行为复用但 ID 独立”的设备，必须同步更新：设备定义、放置分类、图标、精灵、配置面板逻辑与运行时初始化逻辑。

### 5.6 当前已知剩余超时空物品

截至 2026-03-13，本轮已不再登记剩余超时空物品；后续若再次出现，应直接按“未知数据保守处理 + 正式口径优先”规则清理，不新增兼容层。

## 6. 迁移前检查清单

1. 是否已经确认正式目标 ID / 数据 / 资源？
2. 是否真的需要兼容迁移，而不是直接删除未知数据？
3. 是否区分清楚物品资源、设备图标与设备精灵？
4. 是否会影响存档读取、蓝图持久化或导入蓝图？
5. 是否需要为高噪音列表增加默认关闭的辅助过滤开关？
6. 是否已经准备同步回写需求文档、FAQ 与索引？

## 7. 迁移后回归清单

1. 关键物品 / 设备是否还能正常显示；
2. 存档读取时未知物品是否被清空；
3. 存档读取时未知设备是否被删除；
4. 持久化蓝图与导入蓝图是否遵循同样规则；
5. 物品选择器过滤开关的位置、默认值与隐藏范围是否正确；
6. 文档是否已同步更新。
