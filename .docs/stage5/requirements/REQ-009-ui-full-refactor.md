## 背景与目标

当前项目已经积累了多轮功能迭代，界面在布局层级、视觉语言、面板结构、弹窗样式与反馈一致性上存在明显历史痕迹。用户希望将“界面完全重构”升级为 Stage5 的独立需求，以便在不改变业务目标的前提下，系统性重做界面体验，而不是零散修补。

当前新目标进一步明确为：整体界面框架参考 VSCode 的工作台布局思路，形成“顶栏 + 左侧工具条 + 主工作区 + 右侧面板 + 状态栏”的稳定骨架；当前不要求补齐 VSCode 式底部面板。

## 当前口径

1. 本需求是“界面层完全重构”，不是领域逻辑或仿真引擎的全面重写。
2. 重构目标是统一信息架构、视觉样式、交互反馈与组件边界，提升可用性、一致性与后续迭代效率。
3. 核心业务主流程必须保持可用；若某处交互行为需要改变，必须在本需求内明确记录并标注影响面。
4. 当前布局目标参考 VSCode：保留顶部、左侧、右侧、状态栏四段式结构，暂不引入底部面板。
5. 废弃当前左右侧抽屉的激活逻辑，改为在顶栏提供左右面板显式 toggle 入口。
6. 顶栏中的左右面板控制样式参考 VSCode 标题栏中的面板切换按钮：使用并列的小型 toggle button，强调开关态，而不是大尺寸普通按钮。
7. 左侧改为类似 VSCode Activity Bar 的不可关闭工具条，首批固定承载 3 个工具功能：`放置`、`删除`、`蓝图`；与 VSCode 不同的是，每个工具按钮需在图标下方显示中文文字标签。
8. 左侧这 3 个工具按钮不再只是“模式开关”，而是像 VSCode 左栏一样承担 View 切换职责：点击不同按钮后，切换并展示该模式对应的子 Panel 内容。
9. 左侧 View 与现有业务模式保持一一对应，优先复用当前模式切换后的左侧 Panel 内容与事件链路；如无必要，不在首轮重构中改写底层业务语义，仍以“模式切换”为本质。
10. 右侧改为支持折叠分组的面板容器，首批至少支持在同一容器内折叠/展开“基地选择”和“当前选中”两个区域，交互体验参考 VSCode 在 View Container 内的折叠 section。
11. 原百科内容按职责拆分为两个独立入口：`工具箱` 与 `帮助`，不再合并为同一个入口。
12. `工具箱` 继续使用 `dialog` 呈现，承载偏资料与工具的内容；首批固定包含 3 个 tab：`设备一览`、`物品一览`、`产线规划器`。
13. `帮助` 继续使用 `dialog` 呈现，承载偏说明与教学的内容；首批固定包含 3 个 tab：`操作指南`、`进阶说明`、`按键说明`。
14. 当前界面中的长段操作提示文案——“编辑模式：拖拽移动设备，按住框选，按R旋转，按Ctrl+C复制，按住中键移动视角。”——不再直接悬挂在主界面显著区域，应迁入 `帮助` dialog 中的“按键说明” tab。
15. 左侧工具条底部功能区新增 `工具箱`、`帮助`、`设置` 三个入口，作为与顶部“放置 / 删除 / 蓝图”主工具 View 分离的底部功能区。
16. `工具箱` 与 `帮助` 两个入口都从原顶部栏位置迁移到左侧工具条底部，并保持 dialog 打开形态。
17. 左侧 `放置 / 删除 / 蓝图` 三个 View 可进一步细化为更稳定的 section 结构，以提升信息分层、可扫描性与扩展性。
18. 左侧工具条底部新增一个类似 VSCode 的“设置”入口，用于承载少量全局设置项；当前已包含“语言”“超时空模式”和“Ayu Dark / Ayu Light”主题切换。
19. 设置入口打开后使用 `dialog` 呈现，而不是侧边小面板、弹出菜单或独立设置视图。
20. 设置数据应从零散 localStorage key 逐步收敛到单一 `settings` JSON 对象中存储；首版允许通过兼容迁移读取旧 key，再统一写回 `settings`。
21. 当前实现以实际交付为准：设置 UI 已落地，且已支持 `Ayu Dark` / `Ayu Light` 真实切换；默认主题基线仍为 `Ayu Dark`。
22. 界面主题配色可参考 VSCode 的 `Ayu Dark` / `Ayu Light` 风格，优先借鉴其整体层级、明度关系与强调色策略；但不修改画布区域已有业务相关配色与绘制逻辑。
23. 本轮主题改造范围仅限工作台 UI 外壳、面板、按钮、表单、浮层、列表、状态栏等界面层元素，不触及世界画布中的设备着色、覆盖层着色、网格、预览绘制与其他业务可视化颜色口径。
24. 优先覆盖主工作台骨架、顶部栏、左侧工具条、右侧配置区、工具箱、帮助、设置、弹窗/菜单/提示反馈、世界叠加信息等高频界面区域。
25. 允许分阶段落地，但每一阶段都必须避免出现长期不可控的新旧界面混搭。
26. 与界面重构直接相关的底层清理工作，可回链到 `S5-RQ-003`，但界面目标与验收归属本需求。
27. 当前已新增并实现一个文档建档时未单列的新口径：语言设置迁移到左侧底部“设置” dialog 中统一管理。
28. 当前已新增并实现一个文档建档时未单列的新口径：右侧面板在“基地选择”“当前选中”之外，增加独立“电力 / 运行设置”区域，并作为首版工作台的一部分保留。
29. 对“放置”视图内设备分类进一步做二级折叠的建议，当前已被用户明确拒绝；首版继续保持设备分组直接展开展示，不再将“分类可折叠”作为验收要求。
30. 右侧面板当前形态已被用户明确认可；除后续出现新的明确需求外，不再主动追加右侧面板结构性改造目标。
31. 原先“首版主题 UI 落地但不开放真实切换”的限制，已被实际实现提前突破；当前代码已支持 `Ayu Dark` / `Ayu Light` 的真实切换，文档以实际交付为准。
32. 当前已新增一条移动端兼容的检测性需求：当检测到页面视口宽度小于页面视口高度的 $\frac{3}{4}$ 时，在底部状态栏显示“移动页面”；当前阶段仅做状态检测，不进行任何布局调整。
33. 当前已新增一条移动端画布交互需求：支持在主画布上使用双指缩放；该能力只增强移动端可操作性，不构成响应式布局调整。

## 验收标准

1. 主工作流涉及的核心界面区域已统一到同一套布局与视觉语言。
2. 主要高频操作路径在新界面下可完整完成，且无明显效率退化。
3. 顶栏可通过类似 VSCode 标题栏的 toggle button 显式控制左侧与右侧面板显隐，不再依赖旧抽屉激活逻辑。
4. 左侧工具条稳定提供“放置 / 删除 / 蓝图”3 个入口，图标与文字标签清晰可辨。
5. 点击左侧不同工具按钮时，会像 VSCode 左栏切换 View 一样切换到对应模式的子 Panel，而不是仅切换一个裸模式状态。
6. 左侧 3 个 View 的首版内容分配遵循现有左侧 Panel 结构：
	- `放置` View：保留当前“操作区 + 设备分组列表”结构，包括选择/传送带/管道/保存蓝图入口与各设备分类；
	- `删除` View：保留当前“删除模式 + 删除操作”结构，包括单个/整带切换、全部删除、删除全部传送带、清空地块；

10. 左侧工具条底部提供类似 VSCode 的底部功能区，并提供 `工具箱`、`帮助`、`设置` 三个入口，且与主工具 View 分区清晰。
11. `工具箱` dialog 首版固定提供 `设备一览`、`物品一览`、`产线规划器` 3 个 tab，原有核心能力仍可达。

13. 现有“编辑模式”长说明文案已迁入 `帮助` dialog 的“按键说明” tab，主界面不再被该类帮助文本长期占位。
14. 设置入口通过 `dialog` 打开，且与主工具 View 分区清晰。
15. 设置 UI 当前至少提供三个设置项：`语言`、`超时空模式` 与 `Ayu Dark / Ayu Light`。
16. 当前实现口径以实际交付为准：`Ayu Dark` / `Ayu Light` 已支持真实切换，不再视为仅占位或禁用态。

18. 工作台 UI 配色整体可参考 `Ayu Dark` / `Ayu Light` 的层级关系与观感，但不得影响画布业务配色与绘制结果。

## 左侧 View 内容映射

### 放置 View

首版沿用当前 `place` 模式左侧 Panel 的两层结构：

1. 顶部操作区：
	- 选择；
	- 传送带放置；
	- 管道放置；
	- 保存蓝图。
2. 设备区：
	- 按现有设备分组继续展示；
	- 包括物流、资源、存储、基础生产、高级制造、电力、功能、战斗辅助等分组；
	- 保持现有设备选择与放置链路。

建议进一步细化为：

1. 标题区：
	- 显示当前 View 名称“放置”；
	- 显示当前放置状态摘要，如当前选中的操作或设备。
2. 常用操作 section：
	- 选择；
	- 传送带；
	- 管道；
	- 保存蓝图。
3. 设备分类 section：
	- 继续沿用当前设备分组；
	- 保持分组标题直接展示，不再要求二级折叠；
	- 统一采用可滚动列表承载。
4. 当前状态 section：
	- 当前设备；
	- 当前放置方式；
	- 当前旋转等与放置行为直接相关的简要状态。

### 删除 View

首版沿用当前 `delete` 模式左侧 Panel 的两层结构：

1. 删除模式区：
	- 单个删除 / 整带删除切换。
2. 删除操作区：
	- 删除全部；
	- 删除全部传送带；
	- 清空地块。

建议进一步细化为：

1. 标题区：
	- 显示当前 View 名称“删除”；
	- 显示当前删除模式摘要。
2. 删除模式 section：
	- 单个删除；
	- 整带删除。
3. 快捷操作 section：
	- 删除全部；
	- 删除全部传送带；

## 主题配色基线

1. 工作台 UI 外壳配色可参考 VSCode 的 `Ayu Dark` / `Ayu Light`：
	- 深色背景分层清晰；
	- 低饱和中性色作为大面积基底；
	- 暖色或高识别强调色用于激活态、选中态与关键操作。
2. 顶栏、侧栏、面板、状态栏、按钮、输入框、tab、折叠 section、弹窗与菜单等界面层元素应统一到同一套深色主题语言。
3. 首版视觉落地以 `Ayu Dark` 为固定基线；`Ayu Light` 暂只保留为后续主题切换预留项，不要求首版完整实现。
4. 明确排除范围：
	- 不改动画布世界区域中的业务配色；
	- 不改动画布上的设备、网格、覆盖层、预览绘制、物流/管道/电力等业务可视化着色；
	- 不因 UI 主题切换影响已有仿真可视化识别度。
