## 背景与目标

用户需要在仿真已经运行时直接调整能耗或相关参数，并立即观察到设备状态、电力统计与电池变化，而不必重开仿真。

## 当前口径

1. 运行中调整后应即时生效。
2. 调整结果必须反映到仿真状态与电力统计。
3. 若有图表展示，则图表结果必须与底层统计一致。
4. 当前实现口径先采用“全局总耗电覆盖值”：用户输入一个总耗电数值后，不再跟随当前地图设备累计耗电。
5. 交互入口放在右侧面板顶部仿真全局控制区，与“电力模式”同区域，放置态与仿真态都可调整。
6. 输入留空时回退为真实设备累计耗电；界面同时显示真实值作为参考。
7. 仿真运行中修改的覆盖值只对当前这次仿真生效；停止仿真后恢复为进入仿真前的设置值。

## 功能拆解

1. 提供全局总耗电覆盖值输入框，支持空值回退到真实总耗电。
2. 在右侧顶部全局控制区展示真实总耗电、覆盖值与当前生效值。
3. 接通仿真引擎内的运行时更新链路，使覆盖值在运行中修改后直接生效。
4. 同步刷新设备状态、电力统计与电池变化；后续图表需求必须沿用同一底层口径。

## 验收标准

1. 运行中修改能耗后无需重开仿真即可生效。
2. 设备运行反馈与统计结果一致。
3. 总耗电、总发电、净功率、电池变化与底层仿真口径一致。
4. 不因动态修改造成明显卡死、状态错乱或统计滞后。

## bug / 修复记录

### 2026-03-11：富余发电量未回充蓄电池

- 分类：兼容逻辑 / 统计
- 标题：发电恢复后蓄电池不会回充
- 现象：低电阶段电池会下降，但发电恢复后即便出现净正功率，电池也不回升
- 根因：电力结算只处理 deficit 放电，缺少 surplus 回充分支
- 修复：补齐将富余发电量换算为焦耳并写回 `batteryStoredJ` 的逻辑，同时受全局电池容量限制
- 回归：代码已修复并通过构建，待用户联调确认
- 状态：`fixed`
