# S5-RQ-001 缺陷修复与稳定性收敛

## 文档内容范围、结构与格式约定（必读）

### 1. 本文档负责什么

本文档用于沉淀 S5-RQ-001 上的全部结构化记忆，并采用“状态视图 + 时间线附录”的混合写法，只负责本需求相关的：

1. 背景、目标与当前口径。
2. 缺陷清单、优先级、影响范围。
3. 每次需求拉扯后的结论与变更。
4. 验收标准、测试记录、 bug 修复记录。
5. 当前遗留问题与下一步行动。

本文档不负责：

1. 记录无关需求的讨论。
2. 记录逐字聊天内容。
3. 替代总览文档承担整体项目管理。

### 2. 本文档固定结构

1. 文档内容范围、结构与格式约定
2. 需求卡片
3. 背景与目标
4. 当前口径
5. 当前状态视图
6. 验收标准
7. 测试记录
8. bug / 修复记录
9. 中断与恢复记录
10. 需求拉扯与决策时间线
11. 待确认问题
12. 状态更新

### 3. 本文档记录格式

1. 文档前半部分优先回答“现在是什么状态、下一步做什么”。
2. 时间线部分用于记录开发往返、中断、恢复与决策变化。
3. bug 记录使用唯一编号，建议格式：`S5-RQ-001-BUG-XXX`；跨需求 bug 优先同步到 [../bugs.md](../bugs.md)。
4. 测试记录至少写明：时间、场景、结果、证据位置。
5. 若某次讨论只形成“待确认”，写入“待确认问题”，不要伪装成已决策。
