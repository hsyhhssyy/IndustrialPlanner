## 背景与目标

Stage5 需要补齐复杂物流控制能力，使系统可以按物品维度进行准入过滤，并与现有物流判断规则保持一致。

## 当前口径

1. 可以是一个独立设备，也可以是等价能力，但必须能完成物品准入过滤。
2. 必须支持配置目标物品。
3. 非目标物品必须被稳定拒收。
4. 行为要与现有物流系统、前置判定与接收判定口径一致。

## 功能拆解

1. 明确准入口的实现形态与配置入口。
2. 接入现有物品判定与接收逻辑。
3. 在仿真与构建链路中验证过滤结果。
4. 补齐用户可见反馈与错误场景处理。

## 验收标准

1. 可以配置目标物品。
2. 非目标物品会被稳定拒收。
3. 与仿真调度逻辑保持一致。
4. 与现有物流判定无明显冲突。
