## 背景与目标

Stage5 需要补齐复杂物流控制能力，使系统可以按物品维度进行准入过滤，并与现有物流判断规则保持一致。

## 当前口径

1. 可以是一个独立设备，也可以是等价能力，但必须能完成物品准入过滤。
2. 必须支持配置目标物品。
3. 非目标物品必须被稳定拒收。
4. 行为要与现有物流系统、前置判定与接收判定口径一致。
5. 当前已确定最终实现形态为“独立 1x1 新设备”，并归入传送带附属设备类别。
6. 当前已确定该设备为“单入口 + 单出口 + 传送带同速”模型。
7. “数量限制”语义已确认为“累计通过数量上限”，而非“同时驻留上限”。
8. 视觉资源上不需要额外绘制复杂精灵；世界层使用空白图片即可，并且与其他传送带附属设备一样不显示设备名。
9. 实现结构上，`item_log_admission` 不再作为 `conveyor` 特例，而是作为真正的 `junction` 单通道变体。

## 功能拆解

1. 已实现独立设备形态：
	- 1x1；
	- 归入传送带附属设备分组；
	- 单入口、单出口。
2. 已实现运行口径：
	- 未配置准入物品时，行为等同一节普通传送带；
	- 配置准入物品后，仅允许该物品进入；
	- 配置数量后，当前实现为“累计通过数量上限”；
	- 仿真运行时，可在右侧详情查看“当前已通过数量”。
3. 已实现结构口径：
	- 设备归类为 belt family 附属设备；
	- 运行时已改走 `junction` 变体，而非 `conveyor` 变体；
	- 不再依赖 `isBufferedBeltTransportDevice()` 将其作为普通传送带缓存模型处理。
4. 已接入现有物品判定与接收逻辑。
5. 已接入右侧详情配置入口：
	- 准入物品选择；
	- 准入数量上限输入。
6. 已补齐用户可见反馈：
	- 菜单图标入口；
	- 详情面板字段；
	- 正式 PNG 图标与空白世界层精灵。

## 验收标准

1. 可以配置目标物品。
2. 非目标物品会被稳定拒收。
3. 与仿真调度逻辑保持一致。
4. 与现有物流判定无明显冲突。
5. 未配置时，行为与普通传送带一致。
6. 配置数量后，数量限制行为与当前实现口径一致。

## bug / 修复记录

### `S5-RQ-005-BUG-001`

- 编号：`S5-RQ-005-BUG-001`
- 分类：配置/资源
- 标题：首版实现阶段缺少正式 itemicon 与最终世界层资源策略
- 现象：功能最初接通时仍使用临时占位资源，视觉方案未收敛

- 修复：已替换为用户提供的正式 PNG 图标，并将世界层改为空白精灵策略
- 回归：当前轮次用户未反馈相关问题
- 状态：`fixed`

### `S5-RQ-005-BUG-002`

- 编号：`S5-RQ-005-BUG-002`
- 分类：配置/UI
- 标题：物品准入口右侧详情缺少准入物品与数量配置入口
- 现象：设备可像传送带一样运行，但用户在右侧选中详情中看不到设置准入物品和数量的位置
- 根因：配置状态与更新逻辑已接入领域层，但初版未把相关状态和更新函数透传到 `RightPanel`，导致 UI 未渲染
- 修复：已补齐 App 到 `RightPanel` 的透传，新增“准入物品”和“准入数量上限”字段，并补齐文案与设备名称映射
- 回归：构建通过，且当前轮次用户未反馈相关问题
- 状态：`fixed`

### `S5-RQ-005-BUG-003`

- 编号：`S5-RQ-005-BUG-003`
- 分类：语义/UI/资源
- 标题：物品准入口的数量限制语义与资源策略不符合最终口径
- 现象：初版实现把数量限制做成“同时驻留上限”，并临时绘制了 SVG 图标/精灵，且设备名仍可能显示在画布上
- 根因：首版为快速打通链路采用了临时实现，未完全按最终视觉与业务口径收敛
- 修复：数量限制改为“累计通过数量上限”，右侧新增“当前已通过数量”展示；采用用户提供的 PNG 图标；世界层改为空白精灵；设备名按传送带附属设备隐藏
- 回归：构建通过，且当前轮次用户未反馈相关问题
- 状态：`fixed`

### `S5-RQ-005-BUG-004`

- 编号：`S5-RQ-005-BUG-004`
- 分类：结构/运行时
- 标题：物品准入口历史上被实现为 conveyor 特例，而非 belt family 的 junction 变体
- 现象：设备业务语义更接近分流器/汇流器/桥接器等 belt 附属设备，但实现上挂在 buffered belt conveyor 运行模型下，导致结构语义不一致
- 根因：首版为了快速打通功能，直接复用了单通道 conveyor/belt runtime 逻辑
- 修复：已将 `item_log_admission` 重构为真正的 `junction` 单通道变体，并同步调整相关 belt/junction 家族辅助逻辑
- 回归：构建通过，且当前轮次用户未反馈新的结构或行为问题
- 状态：`fixed`
