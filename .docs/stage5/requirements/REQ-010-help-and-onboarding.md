## 文档内容范围、结构与格式约定（必读）

### 1. 本文档负责什么

本文档用于沉淀 S5-RQ-010 上的全部结构化记忆，并采用“状态视图 + 时间线附录”的混合写法，只负责本需求相关的：

2. 面向非开发者、纯用户视角的引导文案、说明书式步骤与快捷键页设计。

1. 替代 `S5-RQ-009` 记录整个工作台骨架重构。
2. 替代业务需求文档记录仿真、蓝图、规划器等领域规则本身。
3. 记录逐字聊天内容。

## 背景与目标

原帮助页虽然已拆分为 `操作指南 / 进阶说明 / 按键说明` 三个 tab，但内容仍偏开发者式摘要，不够适合作为纯用户上手说明。用户明确要求将帮助页改写成更像产品说明书的形式：

1. `操作指南` 面向没有软件开发经验的普通用户；
2. 提到按钮时尽量带图标；
3. 适当引入截图、动图或示意图；
4. `进阶说明` 只放仿真逻辑与高级内容；
5. `按键说明` 改为类似游戏快捷键配置页的只读清单式展示。

## 当前口径

1. 本需求只负责帮助系统与用户引导内容，不负责整个工作台外壳布局。
2. `操作指南` 必须优先回答“我第一步该点哪里、第二步该做什么”。
3. `进阶说明` 可以涉及 tick、缓存、准入口、规划器等机制，但必须避免与用户向步骤混排。
4. `按键说明` 必须是模式 / 功能 / 按键三列式的只读列表风格。
5. 当前允许先用内嵌示意图替代正式截图 / 动图；后续如补素材，应继续归档到本需求。
6. 帮助页在 `Ayu Dark` / `Ayu Light` 下都必须可读。
7. 原本归在 `S5-RQ-009` 的帮助页内容重写与主题适配，自 2026-03-11 起独立迁移到本需求维护；`S5-RQ-009` 仅保留帮助入口与 dialog 容器级职责。

## 验收标准

1. `操作指南` 内容对无开发经验用户可读，不依赖术语背景。
2. `操作指南` 至少说明：主界面分区、左侧工具条作用、左右面板职责、第一次搭产线的推荐顺序。
3. `进阶说明` 只承载高级内容，不与基础步骤混写。
4. `按键说明` 采用清单 / 表格式展示，风格接近游戏快捷键页。
5. 提到主要按钮或模式时，优先配图标或明确视觉标识。
6. 帮助页在深色和浅色主题下均保持可读。
