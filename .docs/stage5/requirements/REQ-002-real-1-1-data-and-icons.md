## 背景与目标

Stage5 需要将现有过渡性内容切换为真实游戏 1.1 版本口径，确保数据、图标与用户可见行为一致，并彻底移除“超时空模式”过渡方案。

## 当前口径

1. 真实 1.1 配方为唯一目标基线。
2. 正式图标应覆盖设备、物品、配方、蓝图等关键展示场景。
3. “超时空模式”应被移除，而不是继续兼容和扩展。
4. 数据切换完成后，相关过渡逻辑应一并治理，避免长期双轨维护。

  2. 建立图标映射清单；

## 实施拆解

1. 建立真实 1.1 配方数据源与当前数据的差异清单。
2. 建立正式图标资源与现有引用点的映射清单。
3. 盘点“超时空模式”相关：
   - 用户入口；
   - 门禁开关；
   - 过渡数据分支；
   - UI 标签与提示文案；
   - 依赖其存在的兼容逻辑。
4. 完成替换后，逐项回归关键界面与仿真链路。

## 验收标准

1. 配方输入、输出、耗时、能耗、适用设备与目标 1.1 版本一致。
2. 正式图标在关键界面展示正确，无明显缺图或错图。
3. 用户侧不再存在“超时空模式”入口、标签或开关。
4. 不再依赖过渡数据分支支撑主流程。
