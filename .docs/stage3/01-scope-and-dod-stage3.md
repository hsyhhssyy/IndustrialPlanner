# Stage3 范围与完成定义（DoD）

## 1. In Scope

### 1.1 电力模型

- 为设备补齐 `powerDemand/powerSupply` 字段。
- 提供设备级与全局级耗电/发电/净功率展示。
- 提供即时值、10 分钟均值、1 小时均值（标准秒）统计。
- 缺电语义改为“不断产”，保持接网前置。

### 1.2 抽水泵效率

- 输出效率从领域模型读取。
- 吞吐结算按效率执行。
- 与外圈放置规则共同验证。

### 1.3 文档口径收口

- 清理 Stage3 内已废弃需求条目。
- 统一反应池口径为“5 共享槽位（已验收）”。

## 2. Out of Scope

- 自动布局、寻路、自动落盘。
- 全局优化求解。
- 复杂物理流体求解。

## 3. DoD

1. `powerDemand/powerSupply` 完整进入领域模型与展示。
2. 缺电场景下设备仍运行，接网前置不被放宽。
3. 电力三窗口统计按标准秒正确滚动。
4. 抽水泵效率可配置并影响输出吞吐。
5. Stage3 文档内不再出现已废弃需求作为验收项。
6. 反应池口径统一为“5 共享槽位（已验收）”。
7. Stage2 冻结能力无 P0/P1 回归。

## 4. 明确废弃项

- 管道等效容量（`2 × 连通长度`）与单液体锁定/排空切换。
- 瓶装液体“隐藏组合 + 专用选择器 + 单展示多展开”。
