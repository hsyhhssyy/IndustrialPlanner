# Stage3 草案：规划器右侧面板新增 Tab（流程图-按设备）技术方案

> 状态：Ready for implementation（口径已拍板）  
> 日期：2026-03-03  
> 目标：在规划器右侧结果区新增第二个 Tab「流程图-按设备」，支持画布拖动节点并以箭头连线。

## 0. 评审结论（2026-03-03 已拍板）

1. 节点数量口径：`xN` 不做统一向上取整，沿用规划计算结果值。
2. 原料节点：显示（无设备的基础供给也作为节点）。
3. 多产物配方：同节点同时显示两个产物，无主次之分。
4. 多产物节点内部结构：两个产物之间使用固定长度直线连接，且排布方向与整体流程方向垂直。
5. 原料箭头锚点：指向上述固定长度直线的中点。
6. 连线标签格式固定：`物品名 45/min · 传送带 x2`。
7. 标签密集策略：优先全显示，允许重叠。
8. 画布交互：必须支持缩放与平移（不仅节点拖拽）。
9. 键鼠平移手势：按住中键拖动进行平移。
10. 需要“重置布局”按钮（恢复自动布局），位置在画布区域左上角。
11. 不做节点位置本地持久化（刷新后不保留）。
12. 节点聚合按 `machineType|recipeId` 全局合并；同配方在不同子链路出现时应合并为同一节点（有意设计）。
13. 连线标签速率按“该边实际分配流量（perMinute）”展示。
14. 首版数据源固定使用“规划结果现成数据（flatCards）”。
15. 本次迭代范围仅交付“流程图-按设备（byBuilding）”，`byProduct` 视角后续迭代。
16. `xN` 展示精度：最多保留 2 位小数，去尾零。
17. 空态文案：`暂无流程图数据`。

## 1. 背景与目标

当前规划器右侧仅有“按层级展开列表”视图。为提升依赖关系可读性，新增并行视图：

- Tab A：展开列表（保留现有实现）
- Tab B：流程图-按设备（新增）

流程图节点定义为“（设备 xN）+ 配方”的唯一组合（全局聚合），并在画布中可拖动；节点之间通过箭头表示物料流向。

## 2. 范围

### 2.1 In Scope

1. 在规划器右侧结果区新增 Tab 切换。
2. 新增“流程图-按设备”画布视图。
3. 节点可拖动，连线实时跟随。
4. 连线使用箭头表示方向。
5. 节点唯一性按“设备 + 配方”聚合（`xN` 仅展示值，不参与唯一键）。

### 2.2 Out of Scope（本草案阶段）

1. 不引入自动布局最优解算法。
2. 不引入第三方图编辑器库（先采用 React + SVG 自研轻量实现）。
3. 不改动规划器求解逻辑与数值结果。

## 4. 视图与交互设计

## 4.1 右侧结果区结构

在结果区标题下方新增 Tab 头：

- 展开列表
- 流程图-按设备

切换仅影响视图，不触发重算；共用同一批规划结果数据。

### 4.2 流程图画布

- 容器：可滚动区域 + 绝对定位图层。
- 节点层：HTML 块（便于文本、图标、交互）。
- 连线层：SVG（path + markerEnd 箭头）。
- 交互：
  - 鼠标按下节点进入拖拽。
  - 拖拽中更新节点坐标。
  - 连线实时重绘。

## 5. 数据建模方案

### 5.1 节点定义

FlowNode：

- id
- machineType
- machineCount
- recipeId
- label
- inputFlows
- outputFlows

节点唯一键建议：

- nodeKey = machineType + "|" + recipeId

说明：

- `xN` 展示不做统一向上取整，沿用规划计算结果值。
- 同设备同配方在不同子链路中会合并为同一节点（有意的全局聚合视图语义）。
- 原料节点（无设备）可使用 RAW + itemId 作为单独键，避免丢失。

### 5.2 连线定义

FlowEdge：

- id
- sourceId
- targetId
- itemId
- perMinute

连线生成逻辑：

- 若源节点 outputFlows 与目标节点 inputFlows 存在同 itemId，则建立 source -> target。
- perMinute 按双方可匹配流量聚合（首版可采用 min(output, input) + item 维度聚合）。

## 6. 自动布局与拖拽策略

### 6.1 初始布局（MVP）

采用分层列布局：

1. 计算每个节点层级（可按入度/BFS）。
2. 按层级分列（从左到右）。
3. 每列按固定间距纵向排布。

### 6.2 拖拽状态

建议状态：

- nodePositions: Record<nodeId, { x, y }>
- draggingNodeId: string | null
- dragOffset: { x, y }

流程：

1. mousedown 记录拖拽节点及偏移。
2. mousemove 更新 nodePositions。
3. mouseup 清理拖拽状态。

### 6.3 画布缩放与平移（必做）

建议状态：

- viewportScale: number
- viewportOffset: { x, y }
- isPanning: boolean

交互：

1. 鼠标滚轮缩放（以指针位置为缩放中心）。
2. 按住中键拖拽进行画布平移。
3. 节点拖拽优先级高于画布平移（避免冲突）。

## 7. UI 展示口径

### 7.1 节点内容

每个节点显示：

- 单产物配方：产物图标 + 产物名称
- 多产物配方：同节点内并列展示两个产物（无主次）
- 下方小字：设备名 x 数量（`xN` 最多 2 位小数并去尾零）

说明：

- 节点语义仍为“设备 + 配方”的唯一组合（全局聚合）。
- 产物名称与图标必须一一对应，禁止显示为设备名或配方名。

多产物配方节点（已拍板）：

- 同一节点内同时展示两个产物（无主次）。
- 若整体流程主方向为左右流，则节点内部采用上下排布（与流程方向垂直）。
- 两个产物之间用固定长度直线连接。
- 原料箭头指向该直线的中点。

### 7.2 连线内容

- 箭头方向：source -> target（强制显示箭头）
- 连线标签：固定展示“该边该物品的实际分配速率 + 对应传送带数量”

标签口径：

- 速率：取该边对该 item 的实际分配流量 `perMinute`（/min）。
- 传送带数量：`ceil(速率 / BELT_THROUGHPUT_PER_MINUTE)`。
- 当前默认带宽口径沿用规划器：`BELT_THROUGHPUT_PER_MINUTE = 30`。
- 标签格式固定：`物品名 45/min · 传送带 x2`。
- 图密集时优先全显示，允许标签重叠。

## 10. 风险与对策

### R1：边数爆炸导致视觉拥挤

- 对策：按 item 聚合边；保持“优先全显示”前提下，仅在极端场景做最小化样式压缩。

### R2：节点唯一键浮点不稳定

- 对策：节点唯一键不依赖 machineCount，避免浮点微差导致重复节点。

### R3：仅用扁平卡片数据可能丢失部分上下文

- 对策：首版固定使用规划结果现成数据（flatCards）；通过验收样例覆盖关键链路，必要时在后续迭代评估切换数据源。

## 11. 验收标准（草案）

1. 右侧结果区可在“展开列表 / 流程图-按设备”间切换。
2. 切换视图不触发规划重算。
3. 流程图节点按“设备 xN + 配方”唯一聚合。
4. 节点可拖动，连线实时跟随。
5. 连线方向正确，箭头可见。
6. 每条连线都显示“该边实际分配速率 + 传送带数量”，并与规划计算口径一致。
7. 标签格式固定为：`物品名 xx/min · 传送带 xN`。
8. 每个节点必须显示产物名称，且与节点图标对应一致。
9. 多产物配方节点可同时显示两个产物，二者由固定长度直线连接；原料箭头指向该直线中点。
10. 支持画布缩放与平移，并提供“重置布局”按钮。
11. 节点位置不做本地持久化；刷新后按自动布局恢复。
12. 空结果时给出空态提示：`暂无流程图数据`。
13. 不影响原展开列表、机器汇总与既有规划流程。
14. 同设备同配方在不同子链路会被合并，且行为符合全局聚合视图设计。
