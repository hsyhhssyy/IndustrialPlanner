# Stage3 回归测试计划

## 1. 测试目标

- 证明 Stage3 新能力可用。
- 证明 Stage2 冻结能力无高优先级回归。

## 2. 测试矩阵

### 2.1 电力模型

1. 全设备 `powerDemand` 显式字段读取与展示正确。
2. 即时净功率计算正确。
3. 缺电时设备阻塞停机（仅运转/停机两态，无降速）。
4. `LOW_POWER` 作为阻塞状态出现条件正确。
5. `OUT_OF_POWER_RANGE` 作为阻塞条件出现条件正确。
6. 标准秒口径与 Stage1 一致（`1 标准秒 = X Tick`，当前实现 `X=20`）。

### 2.2 规划器流程图（按设备）

1. 右侧结果区可切换“展开列表 / 流程图-按设备”。
2. 流程图支持节点拖拽、画布缩放、画布平移、重置布局。
3. 切换流程图视图不触发规划重算。

### 2.3 公共蓝图

1. 可通过 `npm run gen:blueprint-index` 生成索引。
2. 蓝图模式进入时按 `id+blueprintVersion` 增量同步。
3. 本地系统蓝图与索引强一致（含下线删除）。

### 2.4 反应池（回归基线）

1. 反应池维持“5 共享槽位”口径。
2. 并行 lane 运行正常。
3. 路由与出料无回归。

## 3. 验收门槛

- Stage3 新增项：P0/P1 缺陷为 0。
- Stage2 冻结主流程：P0/P1 回归为 0。
- 关键链路（电力/流程图/公共蓝图/反应池）全部通过。
