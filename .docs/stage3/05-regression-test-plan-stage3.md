# Stage3 回归测试计划

## 1. 测试目标

- 证明 Stage3 新能力可用。
- 证明 Stage2 冻结能力无高优先级回归。

## 2. 测试矩阵

### 2.1 电力模型

1. 全设备 `powerDemand` 显式字段读取与展示正确。
2. 即时净功率计算正确。
3. 10 分钟均值在 600 标准秒窗口内正确滚动。
4. 1 小时均值在 3600 标准秒窗口内正确滚动。
5. 缺电时设备阻塞停机（仅运转/停机两态，无降速）。
6. `LOW_POWER` 作为阻塞状态出现条件正确。
7. `OUT_OF_POWER_RANGE` 作为阻塞条件出现条件正确。
8. 设备属性页功率字段展示正确（发电设备 `powerDemand=0`）。
9. 标准秒口径与 Stage1 一致（`1 标准秒 = X Tick`，当前实现 `X=20`），且不受仿真倍速影响。

### 2.2 自动布线

1. 规划器可将当前规划发送到后台任务队列。
2. 队列状态按 `queued -> running -> done | failed | canceled` 流转。
3. `running` 阶段进度上报可观测。
4. `done` 任务仅在“所有连接均正确路由完成”时成立，并产出有效摆放蓝图。
5. 蓝图满足当前基地尺寸边界约束。

### 2.3 反应池（回归基线）

1. 反应池维持“5 共享槽位”口径。
2. 并行 lane 运行正常。
3. 路由与出料无回归。

## 3. 验收门槛

- Stage3 新增项：P0/P1 缺陷为 0。
- Stage2 冻结主流程：P0/P1 回归为 0。
- 关键链路（电力/自动布线/反应池）全部通过。
