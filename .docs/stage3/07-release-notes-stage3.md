# Stage3 发布说明（草案）

## 版本信息

- 阶段：Stage3
- 状态：进行中
- 首次发布日期：待定

## 面向用户的变化

1. 新增电力统计：耗电、发电、净功率，即时/10 分钟/1 小时视图。
2. 缺电行为调整：以当前 `NO_POWER`（阻塞停机）为基线，Stage3 交付目标为拆分为 `LOW_POWER` 与 `OUT_OF_POWER_RANGE`（均阻塞）并移除 `NO_POWER`。
3. 全设备显式补齐 `powerDemand`，设备属性页展示功率字段；发电设备 `powerDemand=0`。
4. 反应池维持既有“5 共享槽位”口径（已验收）。
5. 修复反应池槽位分配残留逻辑，运行时统一按 5 共享槽位分配。
6. 自动布线（Auto Routing）作为 Stage3 必做项，支持后台队列异步生成蓝图。
7. 新增公共蓝图能力（阶段性）：
	- 支持独立命令 `npm run gen:blueprint-index` 生成公共蓝图索引。
	- 蓝图模式进入时异步同步公共蓝图；按 `id+blueprintVersion` 增量拉取正文。
	- 本地系统蓝图与索引保持一致（索引下线项本地同步删除）。
	- 蓝图面板分组为“我的蓝图 / 公共蓝图”，公共蓝图分组常驻并支持空态提示。

## 升级注意事项

- 不引入反应池槽位结构迁移。
- 自动布线任务队列仅会话级保存，关闭浏览器后任务与进度会丢失。
- 公共蓝图索引生成为独立命令，不会自动绑定网站 `build/dev`；新增或更新公共蓝图后需手动执行 `npm run gen:blueprint-index`。

## 已知限制

- 自动布局与全局优化不在本阶段范围内。
- 复杂流体物理特性（压力/温度）不在本阶段范围内。

## 发布后观察指标

- 电力窗口统计稳定性
- `LOW_POWER/OUT_OF_POWER_RANGE` 状态触发准确性
- 反应池回归稳定性
- 自动布线任务队列稳定性与蓝图有效率
- Stage2 冻结主流程回归率
