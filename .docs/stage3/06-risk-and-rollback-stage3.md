# Stage3 风险与回滚

## 1. 主要风险

### R1 电力语义切换风险（高）

- 描述：从“单一 `NO_POWER` 阻塞”切换为“`LOW_POWER/OUT_OF_POWER_RANGE` 双阻塞”存在状态口径迁移风险。
- 缓解：
  - 在 UI 明示当前功率状态。
  - 保留接网前置规则，避免无网运行误解。

### R2 自动布线任务风险（中）

- 描述：自动布线后台任务可能出现积压、状态不一致或超时失败。
- 缓解：
  - 增加队列并发与超时治理策略。
  - 增加状态机流转一致性测试与失败重试机制。

### R3 文档口径漂移风险（中）

- 描述：历史旧口径（如 `NO_POWER`、反应池 3+2）在文档中回流。
- 缓解：
  - 统一以 Stage3 文档为单一口径。
  - PR 检查项加入“状态码与范围口径一致”核对。

## 2. 回滚策略

- 触发条件：任一 P0，或关键链路 P1 未能在窗口内关闭。
- 回滚动作：
  1. 回退至 Stage2 冻结版本。
  2. 自动布线队列为会话级数据，关闭浏览器即丢失；无需执行长期持久化清理迁移。
  3. 重跑 Stage2 回归矩阵。
- 恢复条件：
  - 根因修复并通过 Stage3 + Stage2 双回归。
