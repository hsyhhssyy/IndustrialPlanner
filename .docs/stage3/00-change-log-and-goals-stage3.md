# Stage3 变更总览与目标

## 1. 文档目的

- 明确 Stage3 承接范围与业务价值。
- 建立“需求 -> 设计 -> 测试 -> 发布”追踪主线。

## 2. 输入基线

- Stage2 冻结版本：`Stage2-Frozen-2026-02-26`
- Stage3 启动时间：`2026-02-26`

## 3. 变更摘要（相对 Stage2 冻结版）

### 3.1 新增能力

- 设备功率属性：`powerDemand`、`powerSupply`
- 功率聚合：总耗电、总发电、净功率
- 功率统计窗口：即时值、10 分钟均值、1 小时均值（标准秒）
- 抽水泵输出效率字段（领域模型）

### 3.2 规则改造

- 电力不足不触发停机/降速，仍保留接网前置。
- 抽水泵输出吞吐按配置效率结算。

## 4. Goals

- 补齐 Stage2 未落地的电力与抽水泵口径。
- 让电力链路可观测、可验证、可回归。
- 保持 Stage2 已交付功能稳定，不引入 Stage1 回归。

## 5. Non-goals

- 自动布局、自动连线、全局最优求解器。
- 复杂流体求解（压力/温度/相变）。
- 基地外完整仿真。

## 6. 影响面

- 高：仿真引擎（电力结算）
- 中：UI（电力统计）
- 中：领域模型（设备配置、抽水泵效率）
- 高：回归与兼容验证

## 7. 决策补充（2026-03-01）

> 口径：以下结论以需求决策为准，用于更新 Stage3 活跃范围。

### 7.1 保留为 Stage3 活跃项

- 电力领域模型未引入 `powerDemand/powerSupply`，当前仍为 `requiresPower` 布尔口径。
- 电力不足仍会触发 `NO_POWER` 硬阻塞（停机语义未改造）。
- 未实现电力即时值/10 分钟/1 小时统计窗口。
- 抽水泵尚无“输出效率配置并参与吞吐”口径。

### 7.2 已废弃（不进入 Stage3）

- 管道等效容量（`2 × 连通长度`）与单液体锁定/排空切换。
- 瓶装液体“隐藏组合 + 专用选择器 + 单展示多展开”。

### 7.3 已确认完成

- 反应池已完成验收，正确口径为“5 共享槽位”。

### 7.5 实施补丁（2026-03-01）

- 已修复反应池输入分配残留的“3+2 偏好分配”代码路径，统一为 5 共享槽位分配。
- 已将相关函数名统一为“shared slot pool”语义，避免后续误读。

### 7.4 已由 Stage2 覆盖（不迁移）

- 规划器多目标、地区过滤、配方切换、蓝铁互转条件、农业闭环修正。
- 武陵流体基础设施（管道/分流/汇流/桥接）与固液介质边界。
- 抽水泵外圈限制、武陵建筑上限拦截、水培设备接入、Toast 基础治理。
