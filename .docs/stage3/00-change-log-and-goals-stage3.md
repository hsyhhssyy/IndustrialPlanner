# Stage3 变更总览与目标

## 1. 文档目的

- 明确 Stage3 承接范围与业务价值。
- 建立“需求 -> 设计 -> 测试 -> 发布”追踪主线。

## 2. 输入基线

- Stage2 冻结版本：`Stage2-Frozen-2026-02-26`
- Stage3 启动时间：`2026-02-26`

## 3. 变更摘要（相对 Stage2 冻结版）

### 3.1 新增能力

- 设备功率属性：`powerDemand`、`powerSupply`
- 功率聚合：总耗电、总发电、净功率
- 功率统计窗口：即时值、10 分钟均值、1 小时均值（标准秒）
- 管道系统液体锁定状态与排空状态
- 管道连通长度统计与等效容量计算
- 抽水泵输出效率字段（领域模型）
- 瓶装液体选择器状态（瓶型/液体）
- 反应池独立固液槽位运行时状态

### 3.2 规则改造

- 电力不足不触发停机/降速，仍保留接网前置。
- 管道单液体锁定，异液输入在排空前阻塞。
- 排空判定包含：连通管道本体 + 连通储液罐，不包含设备内部缓存。
- 管道容量口径：`2 × 连通管道本体长度`。
- 抽水泵输出吞吐按配置效率结算。
- 瓶装液体 UI 单展示与执行层多展开保持一致。
- 反应池槽位改为“3 固体 + 2 液体”独立管理。

## 4. Goals

- 补齐 Stage2 未落地高风险规则，消除口径分裂。
- 让电力、流体、瓶装液体三条链路可观测、可验证、可回归。
- 保持 Stage2 已交付功能稳定，不引入 Stage1 回归。

## 5. Non-goals

- 自动布局、自动连线、全局最优求解器。
- 复杂流体求解（压力/温度/相变）。
- 基地外完整仿真。

## 6. 影响面

- 高：仿真引擎（电力/流体结算）
- 中：UI（电力统计、瓶装液体选择器、反应池配置）
- 中：领域模型（设备配置、抽水泵效率）
- 高：回归与兼容验证
